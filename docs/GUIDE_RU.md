## Что такое Clientika

Clientika — это мини-CRM в Telegram для самозанятых специалистов: бьюти-мастеров, репетиторов, тренеров, фотографов и других. Все записи, клиенты, напоминания и учёт дохода — в одном Telegram-боте, без Excel и блокнотов.

### Основные возможности

- Telegram-бот с командами `/start`, `/link`, `/today`, `/help`
- Mini App (React) с экранами: Сегодня, Записи, Клиенты, Услуги, Доход
- Публичная страница записи для клиентов (без авторизации)
- Запись клиента прямо в чате бота (без Mini App)
- Автоматические напоминания клиентам (за 24ч и за 2ч)
- Утренняя сводка записей мастеру
- REST API для Mini App

### Как всё работает вместе

1. **Один процесс** запускает три компонента одновременно:
   - **FastAPI** (HTTP-сервер) — обслуживает REST API и отдаёт статику Mini App
   - **aiogram** (Telegram long-polling) — обрабатывает команды и кнопки
   - **APScheduler** — периодически рассылает напоминания

2. Мастер открывает бота → регистрируется → получает ссылку для клиентов
3. Клиент переходит по ссылке → записывается через Mini App или в чате
4. Мастер получает уведомление → управляет записями в Mini App

## Конфигурация (.env)

Файл `bot/.env` содержит все настройки. Создайте его из `.env.example`:

| Переменная | Обязательна | Описание | Значение по умолчанию |
|---|---|---|---|
| `BOT_TOKEN` | **Да** | Токен Telegram-бота от @BotFather | — |
| `DATABASE_URL` | Нет | Async SQLAlchemy URL | `sqlite+aiosqlite:///./data/app.db` |
| `API_HOST` | Нет | IP адрес для FastAPI | `127.0.0.1` |
| `API_PORT` | Нет | Порт FastAPI | `8000` |
| `WEBAPP_URL` | Нет | Публичный HTTPS-URL Mini App | (пусто) |
| `TELEGRAM_PROXY_URL` | Нет | HTTP/SOCKS прокси для Telegram API | (пусто) |
| `SCHEDULER_INTERVAL_SECONDS` | Нет | Интервал тика шедулера (сек) | `60` |
| `DEFAULT_WORK_START` | Нет | Начало рабочего дня (HH:MM) | `10:00` |
| `DEFAULT_WORK_END` | Нет | Конец рабочего дня (HH:MM) | `20:00` |
| `DEFAULT_SLOT_STEP_MINUTES` | Нет | Шаг слотов (минуты) | `30` |
| `DEFAULT_TIMEZONE` | Нет | Таймзона мастера по умолчанию | `Europe/Moscow` |

### Как получить BOT_TOKEN

1. Откройте [@BotFather](https://t.me/BotFather) в Telegram
2. Отправьте `/newbot` (или `/token` для существующего бота)
3. Следуйте инструкциям — получите токен вида `1234567890:ABCdefGHI...`
4. Вставьте токен в `BOT_TOKEN=` в файле `.env`

### Настройка WEBAPP_URL

Для работы Mini App нужен публичный HTTPS-URL. Варианты:

- **Разработка**: используйте [ngrok](https://ngrok.com/) или Cloudflare Tunnel
  ```bash
  ngrok http 5173  # для dev-сервера Vite
  ```
- **Продакшен**: укажите URL вашего сервера (например, `https://clientika.example.com`)

## Функции для мастеров

### Команды бота

| Команда | Описание |
|---|---|
| `/start` | Регистрация мастера + приветствие + кнопка Mini App |
| `/link` | Показать персональную ссылку для клиентов |
| `/today` | Список записей на сегодня |
| `/help` | Справка по командам |

### Как это работает

1. **Регистрация**: при первом `/start` создаётся профиль мастера с:
   - `display_name` из имени в Telegram
   - `slug` — уникальная строка для ссылки (генерируется из имени)
   - Рабочие часы по умолчанию (10:00–20:00)
   - Шаг слотов 30 мин

2. **Ссылка для клиентов**: формат `https://t.me/<bot_username>?start=<slug>`
   - Клиент переходит → попадает в бота → выбирает способ записи

3. **Кнопка Mini App**: открывает веб-приложение для управления:
   - Записи (создание, редактирование, отмена)
   - Клиенты (база, история визитов)
   - Услуги (добавление, цены, длительность)
   - Статистика (доход за день/неделю/месяц)

### Уведомления мастеру

- **Новая запись**: сразу после записи клиента (имя, услуга, время, стоимость)
- **Утренняя сводка**: в 08:00 UTC — список записей на сегодня

## Запись клиентов

Клиенты могут записаться двумя способами:

### Способ 1: Через Mini App (публичная страница)

1. Клиент открывает ссылку мастера
2. Видит список услуг, выбирает
3. Выбирает день и свободный слот
4. Вводит имя и телефон
5. Подтверждает запись

### Способ 2: Прямо в чате бота

1. Клиент переходит по deep link (`t.me/bot?start=<slug>`)
2. Бот предлагает: "Записаться в приложении" или "Записаться в чате"
3. При выборе "в чате" начинается пошаговый диалог:
   - Выбор услуги (кнопки)
   - Выбор дня (кнопки на 14 дней вперёд)
   - Выбор свободного времени (кнопки-слоты)
   - Ввод имени (текстом или из Telegram-профиля)
   - Ввод телефона (текстом или "Пропустить")
   - Подтверждение записи

### Свободные слоты

Слоты генерируются автоматически на основе:
- Рабочих часов мастера (`work_start_minutes` — `work_end_minutes`)
- Шага слотов (`slot_step_minutes`, по умолчанию 30 мин)
- Уже занятых записей (пересечения исключаются)
- Длительности выбранной услуги

## Mini App (веб-приложение)

### Технологический стек

- React + TypeScript
- Vite (сборка)
- Telegram WebApp SDK (авторизация)

### Экраны

1. **Сегодня** — записи на текущий день
2. **Записи** — полный список (фильтр по дате, статусу)
3. **Клиенты** — база клиентов (поиск, история)
4. **Услуги** — управление услугами (CRUD)
5. **Доход** — статистика за день/неделю/месяц
6. **Публичная запись** (`?master=<slug>`) — страница записи для клиентов

### Аутентификация

Mini App использует стандартный механизм Telegram WebApp:
1. Telegram передаёт `initData` при открытии
2. Фронт шлёт заголовок `Authorization: tma <initData>`
3. Бэкенд проверяет HMAC-SHA256 подпись через `BOT_TOKEN`
4. При первой авторизации создаётся профиль мастера

### Разработка фронтенда

```bash
cd bot/webapp
npm install
npm run dev      # dev-сервер http://localhost:5173
npm run build    # production-сборка в dist/
npm run lint     # проверка ESLint
```

Vite проксирует `/api` запросы на FastAPI (порт 8000).

## REST API

### Базовый URL: `/api`

Все эндпоинты (кроме `/api/public/*` и `/api/health`) требуют авторизации через Telegram initData.

### Эндпоинты

#### Профиль мастера

| Метод | URL | Описание |
|---|---|---|
| GET | `/api/me` | Получить профиль мастера |
| PATCH | `/api/me` | Обновить профиль (имя, slug, рабочие часы, таймзону) |

#### Услуги

| Метод | URL | Описание |
|---|---|---|
| GET | `/api/services` | Список услуг (с параметром `include_hidden=true` — все) |
| POST | `/api/services` | Создать услугу (name, price, duration_minutes) |
| PATCH | `/api/services/{id}` | Обновить услугу |
| DELETE | `/api/services/{id}` | Скрыть услугу (soft delete) |

#### Клиенты

| Метод | URL | Описание |
|---|---|---|
| GET | `/api/clients` | Список клиентов (поиск: `?q=текст`) |
| POST | `/api/clients` | Создать клиента |
| GET | `/api/clients/{id}` | Детали клиента + история записей |
| PATCH | `/api/clients/{id}` | Обновить клиента |
| DELETE | `/api/clients/{id}` | Удалить клиента |

#### Записи (Bookings)

| Метод | URL | Описание |
|---|---|---|
| GET | `/api/bookings` | Список записей (фильтры: `date_from`, `date_to`, `status`) |
| GET | `/api/bookings/today` | Записи на сегодня |
| POST | `/api/bookings` | Создать запись (от мастера) |
| PATCH | `/api/bookings/{id}` | Обновить запись (время, статус, заметки) |
| DELETE | `/api/bookings/{id}` | Удалить запись |

#### Статистика

| Метод | URL | Описание |
|---|---|---|
| GET | `/api/stats?period=month` | Доход, кол-во записей, топ услуг (period: day/week/month) |
| GET | `/api/stats/return-clients?threshold_days=30` | Клиенты, которых пора вернуть |

#### Публичные (без авторизации)

| Метод | URL | Описание |
|---|---|---|
| GET | `/api/public/{slug}` | Информация о мастере + список услуг |
| GET | `/api/public/{slug}/availability?service_id=1&date=2025-01-15` | Свободные слоты |
| POST | `/api/public/{slug}/bookings` | Создать запись от клиента |

#### Health

| Метод | URL | Описание |
|---|---|---|
| GET | `/api/health` | Проверка здоровья сервиса |

### Статусы записей

| Статус | Описание |
|---|---|
| `new` | Новая запись (по умолчанию при записи клиентом) |
| `confirmed` | Подтверждена мастером |
| `came` | Клиент пришёл (визит состоялся) |
| `cancelled` | Отменена |
| `no_show` | Клиент не пришёл |

Активные статусы (показываются в расписании): `new`, `confirmed`, `came`.

## Напоминания и уведомления

### Автоматические напоминания (APScheduler)

Шедулер тикает каждые `SCHEDULER_INTERVAL_SECONDS` (по умолчанию 60 сек) и отправляет:

1. **За 24 часа до записи** → клиенту (если у него есть `tg_user_id`)
2. **За 2 часа до записи** → клиенту
3. **Утренняя сводка** (08:00 UTC) → мастеру (список записей на день)

### Идемпотентность

Каждое отправленное напоминание записывается в таблицу `reminder_states` с ограничением уникальности `(booking_id, kind)`. Повторных отправок не будет даже при перезапуске.

Утренние сводки записываются в `master_daily_summaries` — один раз в день на мастера.

### Надёжность

- Если отправка не удалась (ошибка Telegram) — напоминание НЕ помечается как отправленное
- На следующем тике шедулер попробует снова (в пределах окна)
- Ошибка одного напоминания не блокирует остальные

## Деплой (Docker)

### Dockerfile (multi-stage)

1. **Этап 1** (Node.js): сборка Mini App (`npm run build`)
2. **Этап 2** (Python 3.12): установка зависимостей + копирование webapp dist

### Команды деплоя

```bash
cd bot

# Создать .env с продакшен-настройками
cp .env.example .env
# Заполнить BOT_TOKEN, WEBAPP_URL и т.д.

# Собрать и запустить
docker compose up --build -d

# Посмотреть логи
docker compose logs -f

# Перезапустить
docker compose restart

# Остановить
docker compose down
```

### Важные моменты при деплое

- `API_HOST` должен быть `0.0.0.0` (не `127.0.0.1`) для доступа извне контейнера
- `WEBAPP_URL` — публичный HTTPS-URL вашего сервера
- Данные (SQLite база) хранятся в `./data/` (том `./data:/app/data`)
- Порт 8000 пробрасывается наружу

### Структура в продакшене

```
Сервер (Docker)
├── FastAPI :8000
│   ├── /api/*       — REST API
│   ├── /assets/*    — статика Mini App
│   └── /*           — SPA fallback (index.html)
├── aiogram          — Telegram long-polling
└── APScheduler      — напоминания каждые N секунд
```

## Устранение неполадок

### Бот не отвечает на команды

1. Проверьте `BOT_TOKEN` — правильный ли токен в `.env`?
2. Убедитесь, что бот запущен: в логах должно быть `clientika_start`
3. Проверьте, нет ли другого инстанса, использующего тот же токен
4. Если используете прокси: проверьте `TELEGRAM_PROXY_URL`

### Mini App не открывается

1. Проверьте `WEBAPP_URL` — это должен быть HTTPS-URL
2. URL должен быть доступен из интернета
3. В BotFather: настройте Mini App URL через `/setmenubutton`

### Ошибка "BOT_TOKEN is required"

Файл `.env` не найден или пустой. Убедитесь, что:
- Файл существует в `bot/.env`
- Переменная `BOT_TOKEN` заполнена

### Записи не появляются / слоты пустые

1. Проверьте, добавил ли мастер услуги (без услуг запись невозможна)
2. Проверьте рабочие часы мастера в профиле
3. Проверьте, не занят ли слот другой записью

### Напоминания не приходят

1. Убедитесь, что у клиента есть `tg_user_id` (записался через Telegram)
2. Проверьте, что клиент не заблокировал бота
3. Шедулер работает раз в `SCHEDULER_INTERVAL_SECONDS` — подождите
4. Проверьте логи на ошибки `notifier_send_failed`

### Ошибки при деплое Docker

1. `API_HOST` должен быть `0.0.0.0` для Docker
2. Убедитесь, что порт 8000 свободен
3. Проверьте, что `.env` файл корректен и находится в `bot/`

## Быстрый старт (TL;DR)

```bash
# 1. Клонировать
git clone https://github.com/glebnikonob093-ctrl/selleropus.git
cd selleropus/bot

# 2. Настроить
cp .env.example .env
# Вписать BOT_TOKEN в .env

# 3. Запустить через Docker
docker compose up --build

# Или локально:
python3 -m venv .venv && source .venv/bin/activate
pip install -r requirements.txt
python -m app
```

После запуска:
1. Напишите боту `/start` в Telegram
2. Получите ссылку для клиентов через `/link`
3. Добавьте услуги через Mini App
4. Отправьте ссылку клиентам — они смогут записаться!
